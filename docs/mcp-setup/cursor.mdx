## Installation

Add Storybook MCP to your Cursor MCP settings:

<Steps>
  <Step title="Open Cursor Settings">
    Open Cursor settings (Cmd+, or Ctrl+,) and navigate to **MCP Servers**
  </Step>

  <Step title="Add Server Configuration">
    Click "Add MCP Server" and paste this configuration:

    ```json
    {
      "storybook-mcp": {
        "command": "npx",
        "args": ["-y", "storybook-mcp"],
        "cwd": "${workspaceFolder}"
      }
    }
    ```
  </Step>

  <Step title="Restart Cursor">
    Restart Cursor to activate the MCP server
  </Step>

  <Step title="Verify Installation">
    Open a chat and type: "List all components"

    Cursor should invoke the `list_components` tool.
  </Step>
</Steps>

## Pro License

If you have a Pro license, add it to your configuration:

```json
{
  "storybook-mcp": {
    "command": "npx",
    "args": ["-y", "storybook-mcp"],
    "cwd": "${workspaceFolder}",
    "env": {
      "STORYBOOK_MCP_LICENSE": "your-license-key-here"
    }
  }
}
```

<Tip>
  Or create a `.env` file in your project root with `STORYBOOK_MCP_LICENSE=your-key`
</Tip>

## Multi-Project Setup

For workspaces with multiple projects:

```json
{
  "storybook-mcp-app": {
    "command": "npx",
    "args": ["-y", "storybook-mcp", "--config", "./apps/web/storybook-mcp.config.json"],
    "cwd": "${workspaceFolder}/apps/web"
  },
  "storybook-mcp-admin": {
    "command": "npx",
    "args": ["-y", "storybook-mcp", "--config", "./apps/admin/storybook-mcp.config.json"],
    "cwd": "${workspaceFolder}/apps/admin"
  }
}
```

## Advanced Configuration

### Custom Config Path

Point to a specific config file:

```json
{
  "storybook-mcp": {
    "command": "npx",
    "args": [
      "-y",
      "storybook-mcp",
      "--config",
      "./config/storybook-mcp.json"
    ],
    "cwd": "${workspaceFolder}"
  }
}
```

### Framework Override

Force a specific framework:

```json
{
  "storybook-mcp": {
    "command": "npx",
    "args": ["-y", "storybook-mcp"],
    "cwd": "${workspaceFolder}",
    "env": {
      "STORYBOOK_MCP_FRAMEWORK": "chakra"
    }
  }
}
```

### Debug Mode

Enable verbose logging:

```json
{
  "storybook-mcp": {
    "command": "npx",
    "args": ["-y", "storybook-mcp", "--verbose"],
    "cwd": "${workspaceFolder}"
  }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion icon="circle-exclamation" title="MCP server not starting">
    Check Cursor's output panel:
    1. Open Command Palette (Cmd+Shift+P)
    2. Search for "MCP: Show Logs"
    3. Look for error messages

    Common fixes:
    - Ensure Node.js is installed (`node --version`)
    - Clear npm cache: `npm cache clean --force`
    - Try global install: `npm install -g storybook-mcp`
  </Accordion>

  <Accordion icon="circle-exclamation" title="Tools not appearing">
    Verify MCP connection:
    ```
    @mcp list-tools storybook-mcp
    ```

    Should show all 13 tools. If not, restart Cursor.
  </Accordion>

  <Accordion icon="circle-exclamation" title="License not recognized">
    Verify license key in env:
    ```json
    "env": {
      "STORYBOOK_MCP_LICENSE": "your-actual-key"
    }
    ```

    Or create `.env` file in project root.
  </Accordion>

  <Accordion icon="circle-exclamation" title="Wrong project directory">
    Make sure `cwd` points to your project root:
    ```json
    "cwd": "${workspaceFolder}"  // ← Most common
    "cwd": "${workspaceFolder}/apps/web"  // ← For monorepos
    ```
  </Accordion>
</AccordionGroup>
