## Getting Your License Key

After purchasing, you'll receive your license key via email from Polar.sh:

```plaintext
Your Storybook MCP Pro license key:
polar_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

<Tip>
  Keep this email! You'll need the key to activate Pro features on all your machines.
</Tip>

## Activation Methods

Choose the method that works best for your setup:

<Tabs>
  <Tab title="Environment Variable (Recommended)">
    Add to your shell profile:

    <CodeGroup>
    ```bash .bashrc / .zshrc
    export STORYBOOK_MCP_LICENSE=polar_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    ```

    ```bash .env (project-specific)
    STORYBOOK_MCP_LICENSE=polar_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    ```

    ```powershell Windows PowerShell
    $env:STORYBOOK_MCP_LICENSE = "polar_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
    ```
    </CodeGroup>

    Then restart your terminal or source the file:
    ```bash
    source ~/.bashrc  # or ~/.zshrc
    ```

    **Pros:**
    - Works globally across all projects
    - Doesn't commit secrets to version control
    - Easy to update

    **Cons:**
    - Must configure on each machine
  </Tab>

  <Tab title="Config File">
    Add to `storybook-mcp.config.json`:

    ```json storybook-mcp.config.json
    {
      "framework": "chakra",
      "libraries": [
        // ... your libraries
      ],
      "licenseKey": "polar_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
    }
    ```

    **Pros:**
    - Project-specific configuration
    - Works immediately after clone

    **Cons:**
    - ⚠️ **Do NOT commit this file with the license key**
    - Add to `.gitignore` or use environment variable interpolation
  </Tab>

  <Tab title="MCP Client Config">
    For Cursor/Claude Desktop, add to MCP config:

    **Cursor** (`.cursor/mcp.json`):
    ```json
    {
      "storybook-mcp": {
        "command": "npx",
        "args": ["-y", "storybook-mcp"],
        "cwd": "${workspaceFolder}",
        "env": {
          "STORYBOOK_MCP_LICENSE": "polar_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
        }
      }
    }
    ```

    **Claude Desktop**:
    - macOS: `~/Library/Application Support/Claude/claude_desktop_config.json`
    - Windows: `%APPDATA%\Claude\claude_desktop_config.json`

    ```json
    {
      "mcpServers": {
        "storybook-mcp": {
          "command": "npx",
          "args": ["-y", "storybook-mcp"],
          "cwd": "/absolute/path/to/project",
          "env": {
            "STORYBOOK_MCP_LICENSE": "polar_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
          }
        }
      }
    }
    ```

    **Pros:**
    - Centralized in MCP config
    - Easy to manage multiple projects

    **Cons:**
    - Must configure for each MCP client
  </Tab>

  <Tab title="CLI Flag">
    Pass directly via command line:

    ```bash
    npx storybook-mcp --license polar_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx sync-all
    ```

    **Pros:**
    - One-off usage
    - Testing/debugging

    **Cons:**
    - Must pass every time
    - Visible in shell history
  </Tab>
</Tabs>

## Verify Activation

Check that your license is active:

```bash
npx storybook-mcp check-health
```

You should see:

```plaintext
✅ License: Pro (active)
✅ All Pro features enabled
```

Or programmatically:

```typescript
import { StorybookMCP } from 'storybook-mcp'

const mcp = new StorybookMCP({ /* config */ })
const health = await mcp.checkHealth()

console.log(health.license)
// → { tier: 'pro', active: true }
```

## Troubleshooting

<AccordionGroup>
  <Accordion icon="circle-exclamation" title="License not recognized">
    **Symptoms:**
    - "Pro license required" errors
    - Features showing as unavailable
    - Health check shows "Free tier"

    **Solutions:**

    1. **Verify environment variable is set:**
       ```bash
       echo $STORYBOOK_MCP_LICENSE
       ```
       Should output your license key. If empty, add to shell profile and restart terminal.

    2. **Check for typos in license key:**
       - Should start with `polar_`
       - Exactly as received in email
       - No extra spaces or quotes

    3. **Restart MCP server:**
       - Cursor: Reload window (Cmd/Ctrl+R)
       - Claude Desktop: Quit and reopen app

    4. **Clear cache:**
       ```bash
       rm .storybook-mcp-cache.json
       ```
  </Accordion>

  <Accordion icon="circle-exclamation" title="License validation failed">
    **Symptoms:**
    - "License validation error"
    - Network errors during check

    **Solutions:**

    1. **Check internet connection:**
       License is validated online via Polar.sh API.

    2. **Verify license is active:**
       Log in to [polar.sh](https://polar.sh) and check your purchases.

    3. **Contact support:**
       If license is active but not working, email [support@storybook-mcp.com](mailto:support@storybook-mcp.com) with:
       - Your license key
       - Error message
       - Output of `npx storybook-mcp --verbose check-health`
  </Accordion>

  <Accordion icon="circle-exclamation" title="Works on one machine but not another">
    **Cause:**
    Environment variable not set on new machine.

    **Solution:**

    1. **Option A:** Set environment variable globally (recommended)
       ```bash
       echo 'export STORYBOOK_MCP_LICENSE=polar_xxx' >> ~/.bashrc
       source ~/.bashrc
       ```

    2. **Option B:** Use `.env` file in each project
       ```bash
       # In each project:
       echo 'STORYBOOK_MCP_LICENSE=polar_xxx' >> .env
       ```
  </Accordion>

  <Accordion icon="circle-exclamation" title="License in config file not working">
    **Cause:**
    Config file not being loaded, or syntax error.

    **Solution:**

    1. **Validate JSON:**
       ```bash
       npx jsonlint storybook-mcp.config.json
       ```

    2. **Check config is loaded:**
       ```bash
       npx storybook-mcp --verbose check-health
       ```
       Look for "Config loaded from: ..."

    3. **Use absolute path:**
       ```bash
       npx storybook-mcp --config /absolute/path/to/config.json
       ```
  </Accordion>
</AccordionGroup>

## Security Best Practices

<Warning>
  **Never commit your license key to version control!**
</Warning>

### Protect Your License

1. **Add to `.gitignore`:**
   ```gitignore
   .env
   storybook-mcp.config.json  # if it contains license key
   ```

2. **Use environment variables:**
   Prefer environment variables over config files for sensitive data.

3. **Rotate if compromised:**
   If your key is exposed, contact [support@storybook-mcp.com](mailto:support@storybook-mcp.com) to rotate it.

### Team Setup

For teams, each developer needs their own license. However, you can share config files:

```json storybook-mcp.config.json
{
  "framework": "chakra",
  "libraries": [
    // ... shared config
  ],
  // License loaded from environment variable
  "licenseKey": "${STORYBOOK_MCP_LICENSE}"
}
```

Each developer sets their own environment variable:

```bash
export STORYBOOK_MCP_LICENSE=polar_their_individual_key
```

## CI/CD Setup

Use GitHub Secrets or similar for CI:

<CodeGroup>
```yaml GitHub Actions
name: Storybook Sync
on: [push]
jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: npm install
      - run: npx storybook-mcp sync-all
        env:
          STORYBOOK_MCP_LICENSE: ${{ secrets.STORYBOOK_MCP_LICENSE }}
```

```yaml GitLab CI
sync-stories:
  script:
    - npm install
    - npx storybook-mcp sync-all
  variables:
    STORYBOOK_MCP_LICENSE: $STORYBOOK_MCP_LICENSE
```
</CodeGroup>

Add secret in your CI platform:
- **GitHub:** Settings → Secrets → Actions → New repository secret
- **GitLab:** Settings → CI/CD → Variables → Add variable

<Tip>
  For CI, you can use the same license key as your team members. CI usage doesn't count toward per-developer limits.
</Tip>

## Transferring License

Need to transfer your license to a new email or organization?

1. Email [support@storybook-mcp.com](mailto:support@storybook-mcp.com)
2. Provide:
   - Current license key
   - Original purchase email
   - New email address
3. We'll transfer within 24 hours

<Info>
  License transfers are free and unlimited.
</Info>

## Upgrading from Free

Already using the free tier? Your existing projects will automatically use Pro features once activated:

1. Add license key (any method above)
2. Restart MCP server
3. All Pro features immediately available
4. Re-sync to use new templates:
   ```bash
   npx storybook-mcp sync-all --overwrite
   ```
